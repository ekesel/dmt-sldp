Holiday Calendar API

Permissions

· POST / PATCH → Only manager

· GET (all / single) → Any authenticated user

---

1.Upload Holiday Calendar (POST)

Endpoint

POST https://api.elevate.samta.ai/api/homepage/holiday-calendar/

Headers

Authorization: Bearer <access_token> Content-Type: multipart/form-data

Body (form-data)

holiday_calendar_file: <file>

Permission

· Non-manager → Error

· Manager → Allowed

Success Response

{ "message": "Holiday calendar uploaded successfully" }

Error Example

{ "error": "Only manager can perform this action" }


2. Update Holiday Calendar (PATCH)

Endpoint

PATCH https://api.elevate.samta.ai/api/homepage/holiday-calendar/{id}/

Body

holiday_calendar_file: <file>

Permission

· Only manager

3. Get All Holiday Calendars

Endpoint

GET https://api.elevate.samta.ai/api/homepage/holiday-calendar/

Access

· Any user

Response

[ {

"id": 1, "holiday_calendar_file": "/media/Samta/holiday_calendar/dmt_terms_version_2.pdf" } ]



4. Get Single Holiday Calendar

Endpoint

GET https://api.elevate.samta.ai/api/homepage/holiday-calendar/{id}/

Example

GET https://api.elevate.samta.ai/api/homepage/holiday-calendar/1/

Response

{ "id": 1, "holiday_calendar_file": "/media/Samta/holiday_calendar/dmt_terms_version_2.pdf" }

1. Onboarding API 

Base URL: /api/homepage/onboarding/  

Detail URL: /api/homepage/onboarding/<id>/  

Permission: IsManagerOrReadOnly  

Content-Type: multipart/form-data  

Managers can create, update, and delete records. Other users have read-only access. 

Shape 

A. List Onboarding Documents 

Method: GET  

URL: /api/homepage/onboarding/  

Request Body: None  

Response — 200 OK 

[ 
  { 
    "id": 1, 
    "title": "Employee Handbook", 
    "onboarding_file": "http://example.com/media/tenant/onboarding/handbook.pdf" 
  }, 
  { 
    "id": 2, 
    "title": "IT Policy", 
    "onboarding_file": "http://example.com/media/tenant/onboarding/it_policy.pdf" 
  } 
] 

Shape 

B. Retrieve Single Onboarding Document 

Method: GET  

URL: /api/onboarding/<id>/  

Request Body: None  

Response — 200 OK 

{ 
  "id": 1, 
  "title": "Employee Handbook", 
  "onboarding_file": "http://example.com/media/tenant/onboarding/handbook.pdf" 
} 

Response — 404 Not Found 

{ 
  "message": "Onboarding document not found" 
} 

Shape 

C. Create Onboarding Document 

Method: POST  

URL: /api/homepage/onboarding/  

Request Body (FormData) 

Field                 Type          Required 

title               String          Yes 

onboarding_file      File          Yes 

Example 

title: Welcome Kit 
onboarding_file: (Binary File) 

Response — 201 Created 

{ 
  "message": "Onboarding document uploaded successfully" 
} 

Response — 400 Bad Request 

{ 
  "title": ["This field is required."], 
  "onboarding_file": ["No file was submitted."] 
} 

Shape 

D. Update Onboarding Document (Partial Update) 

Method: PATCH  

URL: /api/homepage/onboarding/<id>/  

Request Body (FormData) 

Field               Type    Required 

title               String     No 

onboarding_file     File      No 

Response — 200 OK 

{ 
  "message": "Onboarding document updated successfully" 
} 

Shape 

E. Delete Onboarding Document 

Method: DELETE  

URL: /api/homepage/onboarding/<id>/  

Request Body: None  

Response — 200 OK 

{ 
  "message": "Onboarding document deleted successfully" 
}
